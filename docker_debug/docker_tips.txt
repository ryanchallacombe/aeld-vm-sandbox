I ran into the same issue:  mknod: dev/null: Operation not permitted
Yet my local run of full-test.sh passed. After spending hours debugging, after seeing this post from 'Alessandro Masini' I suspected my docker configuration was the issue. 
I was running in the wrong docker context. To see the available contexts run >> docker context list
Mine showed a 'default' and 'rootless' context. Make mknod issue resolved when I switched from rootless to default. 
To see the currently running context run >> docker info
Near the top of the command output it will list the context. To change context run >> docker context use <context_name>
In my case, I did >> docker context use default
Next, restart docker with >> sudo systemctl restart docker
